# BLOCK RUSH - KINAKO MIX 仕様書 v2

> Tropical Dream Club リリース告知用 スマホブラウザミニゲーム
> 8bit × ドリーミー × ビーマニオマージュ × 幽白スーファミテイスト

---

## 1. プロジェクト概要

### 目的
楽曲リリースの告知＆ダウンロード誘導を兼ねた、ブラウザ上で動くインタラクティブな音楽体験ミニゲーム。プレイヤーは曲を最後まで聴きながら、5鍵パッドでブロック崩しを楽しむ。

### ターゲット環境
- スマートフォンの縦画面ブラウザ（iOS Safari / Chrome / Android Chrome）
- 横向きには対応せず、横にされたらタイトルへ戻す
- デスクトップでも動作するが、モバイル最適化を優先

### デプロイ先
- **GitHub Pages**（同一リポジトリで楽曲ZIPも配信）
- 静的サイトのみ、サーバー処理は一切なし

## 2. ビジュアルデザイン方針

### 全体トーン
- **8bit × ドリーミーポップ × 幽白スーファミ風**
  - ファミコン/ビーマニ筐体の物理感
  - Tropical Dream Clubのカラフル世界観
  - スーファミ『幽☆遊☆白書』のメカニカル装飾フレーム
- ピクセルアート風、ローテク感、CRT風スキャンライン
- `image-rendering: pixelated` でカクカク感を維持

### 円形ステージ背景（重要）
ゲームプレイ画面の**ブロックゾーンの背景は円形（リング）状**。スーファミ幽白の試合会場のような、円形の闘技場っぽいイメージ。
- 中央が暗く、周囲に向かって明るくなる**放射グラデーション**
- ブロックが闘技場の中で浮かんでいるような構図
- 円形の輪郭にうっすらと装飾枠（ピクセル風）

### カラーパレット
| 用途 | 色コード |
|---|---|
| 背景グラデーション | `#1a0a2e` → `#2d1b4e` → `#4a1a4e` |
| メインピンク | `#ff6b9d` |
| メインイエロー | `#ffd93d` |
| メイングリーン | `#5DCAA5` |
| メインパープル | `#c8b6ff` |
| 装飾フレーム背景 | `linear-gradient(180deg, #4a1a4e, #2d1640)` |
| OFF状態（ステム） | `#555` / `#222` |

### フォント
- **Press Start 2P**（Google Fonts）または `Courier New, monospace` フォールバック
- すべてピクセルフォント想定、サイズは小さめ（7px〜14px中心）

### 浮遊絵文字
- ★ ♪ ◆ ▲ ● ◇ をピクセル風記号として使用
- 4色（ピンク／イエロー／グリーン／パープル）で散らす
- ふわふわ float アニメーション（8〜14秒ループ、ランダムディレイ）
- ブロックゾーン内および背景全体に配置

### 観客席風ドットノイズ帯（最上段装飾）
- 画面最上部に高さ約60〜80pxの帯
- カラフルなドットがランダムに配置されたピクセルノイズ
- スーファミ幽白の客席を彷彿とさせる演出
- 下端は背景にフェードアウト

### キャラクター応援要員（賑やかし）
- きなこ・ぶん（butterfly-shooter と共通キャラ）を画面端に配置
- 素材は `assets/characters/kinako.png` / `bun.png` に格納済み
- 観客席エリアか画面下端の応援ポジションでふわふわ揺らす想定
- 実装は Phase 3（ビジュアル仕上げ）で対応

### 3-1. ローディング/タイトル画面

**表示要素（上から順）**
1. **観客席風ドットノイズ帯**（最上段）
2. **ブランドロゴ**：`TROPICAL DREAM CLUB`（小さめ・ピクセルフォント）
3. **ゲームタイトル**：`BLOCK RUSH` / `- KINAKO MIX -`（大きめ・3D影付き）
4. **メッセージウィンドウ風警告ボックス**
   - 4隅にイエロードット（スーファミ風枠）
   - タイトル：`『 PRESS START 』`
   - 注意事項：
     - `▸ TURN OFF SILENT MODE`
     - `▸ HOLD PHONE VERTICALLY`
     - `▸ USE HEADPHONES`
5. **ロードバー**（4色レインボーパターン）＋進捗%
6. **プレス可能領域**：画面全体タップでゲーム開始

**動作**

- ロード完了後、自動的にゲームプレイ画面へ遷移

### 3-2. ゲームプレイ画面（縦画面）

**レイアウト（上から順）**

#### ① 観客席風ドットノイズ帯（高さ約40〜60px）
- ローディング画面と共通、画面トップの装飾

#### ② HUD（高さ約24〜28px）
- 左：`SCORE 04720`（イエロー、ピンク影）
- 右：`02:34 / 04:12`（経過時間 / 楽曲総尺、パープル）
- **楽曲進捗バー**：HUDの直下に細い横バー（4色グラデーション、左から伸びる）
- ※「技」「霊」エンブレム＋HPバー風UIは**削除**（意味不明なため）

#### ③ ブロックゾーン（高さ約200px）
- **背景：円形ステージ**（放射グラデーション、リング装飾）
- ブロック群（4色×8列）が円形ステージの上半分に浮遊
- 弾は鍵の真上から発射され、円形ステージ内で乱反射
- 浮遊絵文字をブロックゾーン内に散らす
- 最下端に細い虹色ライン（電飾オマージュ）

#### ④ ステムスイッチパネル（高さ約44px）
- 装飾フレーム（パープル基調）で囲む
- **ロッカースイッチ風**4つ：BEAT / BASS / SYNTH / VOCAL
- 各スイッチごとに対応色（ピンク / パープル / グレー / イエロー）
- ON＝上位置、OFF＝下位置（即ミュート）

#### ⑤ 5鍵パッド（高さ約100px）
- **ビーマニ正配列**（重要）：
  - **下段に白鍵3つ**（鍵1・3・5）
  - **上段に黒鍵2つ**（鍵2・4）、それぞれ白鍵と白鍵の間に配置
- 白鍵：白〜薄グレーグラデーション、ピンク縁、ピンク影
- 黒鍵：濃紺〜黒グラデーション、パープル縁、パープル影
- 各鍵に番号（1〜5）薄く表示
- 装飾フレーム（パープル基調）で囲む

## 4. 音響仕様

### 楽曲ファイル
- **形式**：Opus（メイン） + MP3（フォールバック）
- HTML5 `<audio>` タグの multi-source 指定で自動切替
- ステム数：4（beat / bass / synth / vocal）
- 全ステムを `AudioBufferSourceNode` で完全同期再生

### ステム再生制御
- 各ステムは独立した `GainNode` を持ち、スイッチで `gain.value` を 0/1 で切替
- **即ミュート**（フェードなし、パチッと切替）

### ビート解析（ブロック生成トリガー）
- BEATステムに `AnalyserNode` を接続
- 低周波域（〜150Hz）の音量を10ms間隔でサンプリング
- 閾値（動的調整）を超えた瞬間にキック検出
- 検出時：ランダムなレーンに新規ブロック追加（既存ブロックは1段下にスライド）

## 5. ゲームロジック

### 5-1. 5鍵パッドと弾発射
- 5つの鍵に対応する5つの発射ポジション（鍵の真上中央）
- タップ/クリックで該当ポジションから**真上に弾を発射**
- 同時押し可能（マルチタッチ対応）
- 弾の寿命：3秒（生成から3秒で消滅）

### 5-2. 弾の物理挙動
- 初速：垂直上向き（速度約400px/sec）
- ブロック・壁・天井に当たると**反射**（円形ステージの境界でも反射）
- 反射方向は入射角の法則に従う（左右の壁／天井／ブロック上下面）
- 一発の弾で複数ブロック破壊可能

### 5-3. ブロック仕様
- 初期配置：8列×4段（計32個）、4色グラデーション（ピンク→イエロー→グリーン→パープル）
- ピクセルベベル風のドット絵スタイル
- キック検出ごとに新規ブロックがランダムレーンに生成され、既存ブロックは1段下にスライド
- ブロックがプレイヤー側ライン（5鍵の上端）に達してもペナルティなし（勝敗無し）

### 5-4. 演出
- 弾がブロックに当たった瞬間：
  - パーティクル飛散（ブロックと同色のドット）
  - 効果音（8bit風ヒット音）
- 崩したブロック数に応じてビジュアル派手化：
  - 100個以上：背景にうっすら追加グラデーション
  - 200個以上：浮遊絵文字の数増加
  - サビ突入時：画面全体フラッシュ

### 5-5. ゲーム終了条件
- 楽曲がフル尺再生終了 → **STAGE CLEAR画面へ自動遷移**
- 勝敗・ゲームオーバーは存在しない

## 6. 入力・端末対応

### 6-1. マルチタッチ
- 5鍵パッド＋ステムスイッチで同時操作可能
- `touchstart`/`touchend` イベントを使用、`pointer-events` 適切に設定

### 6-2. 端末向き検知
- `window.matchMedia('(orientation: portrait)')` で検知
- 横向き → 全画面警告オーバーレイ
- ゲーム中に横向きにされた場合 → タイトル画面に戻す（楽曲・状態リセット）

### 6-3. オーディオ初期化
- iOS Safariの`AudioContext.suspended`問題への対応
- 必ずユーザータップ後に `AudioContext.resume()` を呼ぶ
- ローディング画面のタップを起点にする

- ZIP中身：（実装フェーズで決定）
  - WAVステム4本（DJ/リミックス向け）または

### ダウンロードリンク
- `<a href="./download/xxx.zip" download>` で直接ダウンロード
- ゲーム終了画面のメインCTAとして配置

## 8. 技術スタック

### フロントエンド
- **Vanilla JS** または **軽量フレームワーク**（プレーンHTML/CSS/JS推奨）
- `<canvas>` でゲームエリア描画（ブロック・弾・パーティクル・円形ステージ）
- HTML/CSSでUI（HUD・スイッチ・パッド・装飾フレーム）

### 音声処理
- **Web Audio API**（`AudioContext`, `AudioBufferSourceNode`, `GainNode`, `AnalyserNode`）
- ライブラリ不要、ネイティブAPIで実装

### ファイル構成例
```
/
├── index.html          # ローディング画面＋ゲーム本体（SPA）
├── style.css
├── js/
│   ├── audio.js        # 音響制御・ステム同期
│   ├── game.js         # ゲームループ・物理演算
│   ├── ui.js           # UI制御（スイッチ・パッド）
│   ├── analyzer.js     # ビート解析
│   └── stage.js        # 円形ステージ描画
├── audio/
│   ├── beat.opus / .mp3
│   ├── bass.opus / .mp3
│   ├── synth.opus / .mp3
│   └── vocal.opus / .mp3
└── download/
    └── tropical_dream_club_kinako_mix.zip
```

## 10. 実装メモ（Claude Codeへの指示）

### 重要事項
1. **ファイル名はすべて英数字**（日本語非推奨、特にダウンロードZIP）
2. **iOS Safari対応**：AudioContext起動はユーザータップ必須
3. **モバイル最重視**：縦画面前提、タッチ操作前提
4. **GitHub Pages配信前提**：絶対パス禁止、相対パス使用
5. **8bit美学**：角丸禁止（直角ベース）、`image-rendering: pixelated` 必須
6. **5鍵配列はビーマニ正準拠**：下段に白鍵3つ、上段に黒鍵2つ（白鍵の間）
7. **HPバー風の装飾UIは入れない**（HUDは進捗バーのみシンプルに）

### 段階的実装推奨
最初は仮の音声ファイル（無音WAV4本）でロジックを組み、後から本物のステムを差し替える流れ。
